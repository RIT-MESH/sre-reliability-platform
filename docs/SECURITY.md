# Security

## Threat model (summary)

- **Credential leakage**: mitigated by no hard-coded secrets, git-ignored
  `.env`/state/keys, Secrets Manager at runtime, GitHub OIDC.
- **Exposure of data tier**: DB/Redis only reachable from the app SG; no public
  accessibility; no 0.0.0.0/0 on data tiers.
- **Compromised instance**: IMDSv2 enforced, least-privilege IAM, SSM access.
- **Supply chain**: dependency, container, Terraform and secret scanning in CI.
- **Data at rest / in transit**: S3/RDS/ElastiCache encrypted; Redis TLS; ALB TLS.

## Credentials

- No real credentials are committed. `.env.example` contains only fake values.
- AWS: **GitHub OIDC** → short-lived role assumption; no long-lived access keys
  in GitHub secrets.
- DB credentials: generated by Terraform (`random_password`) and stored in
  **AWS Secrets Manager**; the app reads them via IAM at runtime. The password
  is referenced by ARN, not embedded in source or user-data.
- Redis AUTH token: passed as a sensitive variable (`TF_VAR_redis_auth_token`),
  sourced from Secrets Manager in a real pipeline.

## Network security

| Tier | Inbound | Source |
| --- | --- | --- |
| ALB | 80/443 | internet |
| App | 8000 | ALB SG + VPC (Prometheus) |
| RDS | 5432 | app SG only |
| Redis | 6379 | app SG only |

## IAM (least privilege)

- EC2 instance role: SSM core + CloudWatch agent + Secrets/SSM read + logs/metrics
  write. Broad `*` resources are used where scoping requires ARNs you supply;
  tighten to specific ARNs/tags in production (noted in module comments).
- GitHub Actions role: scoped to the state bucket + lock table + STS identity;
  attach additional least-privilege policies for `apply` in a real account.
- Monitoring role for RDS enhanced monitoring (prod).

## Encryption

- EBS volumes encrypted; S3 SSE-KMS; RDS storage encrypted + KMS; ElastiCache
  encryption at rest + TLS in transit.

| Container | Trivy | `docker.yml` |
| Terraform | tfsec | `terraform.yml` |
| Private keys | pre-commit detect-private-key | pre-commit |

## Operational hygiene

- `.gitignore` excludes `.env`, `*.tfstate`, `.terraform/`, `*.pem`, `*.key`.
- `pre-commit` runs formatting, validation and secret/key detection locally.
- Deletion protection + final snapshots (prod) prevent accidental data loss.
